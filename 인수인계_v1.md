### 3-2. 통합 결함 수정 (모두 js/main.js, Fable 검수에서 발견)
1. **createSession.onPlaced — 배치 기구와 시각화 계층 단절(치명)**: interact.js는 기구를 스냅 디스크의
   부모인 markers 그룹에 붙이는데, idealgas의 `findEquipment(anchor)`는 **anchor 직계 자식**에서
   `userData.equipment`로 탐색 → 영원히 못 찾아 입자·피스톤·센서가 전부 침묵하는 구조였음.
   수정: onPlaced에서 `obj.userData.equipment = {expId, itemId, slot}` 태깅 + `bench.anchor.attach(obj)` 재부착
2. **pick 핸들러 — 조립 판정 미갱신**: 배치된 기구를 다시 들어도 checkAssembly가 재실행되지 않음.
   수정: pick 이벤트에서 `session.refreshAssembly()` 호출 + 집는 순간 `userData.equipment = null`
3. **세션 자원 누수·선반 복귀**: 세션 종료 시 실험대에 남은 기구가 dispose 안 됨 / 실험대에서 집은 기구를
   "선반 탭"으로 되돌리면 실험대 위치로 돌아가는 문제. 수정: 최초 픽업 시 `__shelfOrigin` 저장 후 복귀는
   항상 선반으로, dispose는 세션이 만든 stocked 메쉬 전체를 부모 무관하게 제거+dispose
- 부수: `goSpace()`에 `controls.setGroundFn(handle.getGroundY || null)` 연결(자연 공간 지형 보행)

## 4. 트러블슈팅 이력

### 4-1. Sonnet 에이전트 세션 한도 중단
- 병렬 투입한 A·B 에이전트가 중간에 사용량 한도로 강제 종료(파일 일부만 생성). SendMessage 도구가 없어
  이어받기 불가 → 잔여 파일 목록을 정확히 산출해 **A-2·B-2 에이전트를 새로 투입**, 기존 코드 Read 후
  동일 스타일로 완성시킴. B-2가 보고한 실계약 확정 사항은 §6.6 참조
- 교훈: 대형 병렬 에이전트 투입 전 사용량 잔량 확인, 계약 파일을 먼저 고정해 두면 중단돼도 이어받기 쉽다

### 4-2. 샌드박스 npm 403 차단
- esbuild/jsdom/three/ws 설치 불가 → 의존성 없는 자체 검증(임포트 그래프 스크립트 + node --check)으로 대체.
  이 때문에 `server/relay.js`는 구문 검사만 통과, **실구동 미검증**(§6.4)

### 4-3. 작업트리 파일 잘림 사고 (이번 세션 말미, 원인 미확정)
- 증상: main.js·nature.js·config.js·ROADMAP.md 4개 파일의 꼬리가 중간 토큰에서 잘림
  (예: main.js가 `get ready() { r`로 끝남). 도구별 읽기 결과도 불일치(동기화 스냅샷 문제 정황)
- 진단: `git diff`로 HEAD 대비 삭제만 감지 + `wc -c` 크기 비교로 확정. **HEAD(=origin/main)는 전부 정상**이고
  사용자의 Supabase 키 입력도 커밋에 포함돼 있었음
- 조치: `git checkout -- <4개 파일>` 복원 → HEAD와 바이트 단위 일치, 구문 OK, 테스트 ALL PASS 재확인
- 교훈: **이 폴더(Documents 하위)는 동기화로 파일이 훼손될 수 있다. 커밋을 자주 하고, 세션 시작·종료 시
  `git status` + 의심 파일은 `wc -c`로 HEAD와 크기 비교.** 잘림 재발 시 원인 조사 필요(§6.8)

## 6. 미해결·이월 과제

### 6.1 [new, core] 브라우저 런타임 미검증
정적 검증(구문·임포트·물리 테스트·계약 대조)만 완료. 실제 WebGL 렌더·터치 조작·HUD 동작은 브라우저에서
한 번도 실행 안 됨. 통합 결함 수정(§3-2)이 정적 대조 기반이므로 런타임 오류가 남아 있을 수 있다. 최우선.

### 6.2 [new, core] GitHub Pages 활성화 여부 미확인
push는 완료. Pages 설정은 이 세션에서 확인 불가였음(사용자 GitHub 설정 화면 필요).

### 6.3 [new, core] Supabase 실연결 미검증 + 트래픽 한도 운영 규칙
키는 커밋됐으나 실제 channel subscribe 성공 여부 미확인. 운영 규칙: **룸당 10명 이하(모둠별 방 코드)**.
40명 단일 룸 상시 사용 시 무료 티어 월 200만 메시지 한도 초과 위험(산정 근거: ARCHITECTURE §3).
대규모 상시 사용이 필요해지면 wsrelay(교내망) 또는 Cloudflare PartyKit 이전(백로그 E).

### 6.4 [new, minor] server/relay.js 실구동 미검증
샌드박스 npm 차단으로 `ws` 설치 불가였음. 교사 PC에서 `npm install ws && node server/relay.js` 후
정적 서빙(:8080)과 wsrelay.js 프로토콜(`{t:'join'|'state'|'peers'|'leave'}`) 상호 동작 확인 필요.

### 6.5 [new, minor] 그래프 등온선 참조곡선 미구현
AGENT_BRIEF B-4의 "등온선 참조곡선 점선"이 B-2 재투입 명세에서 누락돼 빠짐. graph.js에
`setReference(fn)` 추가 + P–V 탭에서 현재 T의 등온선 점선 오버레이(구현 5~10분 규모).

### 6.6 [new, minor] registry.js 주석 스키마 ≠ 실제 ui 계약
실제 계약(= idealgas/index.js가 정의하고 hud.js가 구현한 것)은 registry.js 주석보다 확장돼 있다:
control `type:'stepper'` 존재, `requiresItem`(control·option 단위 활성 조건), `activeWhen:{bind,equals}`,
`momentary:true`(놓으면 0 복귀). **새 실험 구현 시 registry.js 주석이 아니라 idealgas/index.js를 참조할 것.**
registry.js 주석 갱신 권장.

### 6.7 [new] 백로그 C(라울 법칙)·D(반응 속도) 구현 대기
스텁 파일 상단에 모델 수식·장비·UI·검증 시나리오 설계 주석 완비
(`js/experiments/raoult/index.js`, `js/experiments/kinetics/index.js`). E(공동 실험 동기화·교사 대시보드)·
F(보고서/CSV)는 docs/AGENT_BRIEF.md 백로그 참조.

### 6.8 [new, core] 파일 잘림 재발 감시
§4-3 사고의 원인(OneDrive류 동기화 vs 세션 마운트) 미확정. 재발 시 어느 도구가 어떤 크기로 읽는지 기록하고,
복구는 항상 `git checkout --`(HEAD가 신뢰 원본).
