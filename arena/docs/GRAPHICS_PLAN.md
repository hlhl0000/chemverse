# ChemVerse ARENA — 그래픽 개선 방안 (Phase G 제안, 설계: Fable)

> 사용자 요구: ① 실험 도구를 이름표가 아닌 "게임 아이템스러운 + 실물 특징 반영" 그래픽으로
> ② 게임 맵·배경 구체화. — ①은 **Phase B에서 이미 구현 완료**(items.js 부품 6종·무기 4종 실물 메쉬,
> 크레이트 부유·회전 연출). 본 문서는 ②와 후속 폴리싱의 실행 계획이다. 승인 후 착수.

## 0. 제약 (전 항목 공통 — 위반 금지)

- 성능 예산 유지: 텍스처 금지(버텍스 컬러·절차적 지오메트리만), 그림자맵 금지, 드로우콜 ≤150,
  InstancedMesh 우선, pixelRatio ≤1.5, 보급형 태블릿 30fps
- 시드 결정성: 배치·변형은 전부 mulberry32(seed) — 전 클라이언트 동일
- 담당: A형 에이전트(arena.js·voxel.js·effects.js·신규 theme.js) / Fable 검수

## G-1. 맵 테마 — "옥상 개방형 화학 실험동" (저비용·고효과, 1순위)

현재 맵은 회색 바닥+추상 블록. 이를 의미 있는 화학 공간으로:

- **바닥**: 대형 평면 1장 → 세그먼트 분할 + 버텍스 컬러로 2톤 타일 체커(#13161f/#1a1f2b),
  중앙 격전지는 육각 경고 패턴(주황 라인), 팀 진영 앞은 팀색 유도선. 드로우콜 1 유지
- **외벽**: 단순 벽 → 안전 펜스 프레임 + 상단 경고 스트라이프(흑/황 버텍스 컬러 줄무늬),
  모서리 4곳에 배기 덕트 타워(실루엣용 수직 요소)
- **스카이 돔**: SphereGeometry(backside) 버텍스 컬러 그라데이션 — 남색(천정)→노을 주황(지평),
  지평선에 학교 건물 스카이라인 실루엣 평면 4장(원경, 안개에 잠김). fog 색을 지평 색과 일치

## G-2. 엄폐물 → 실험실 오브젝트 교체 (2순위)

추상 블록 InstancedMesh를 저폴리 실험실 가구 5종 풀로 교체(종류별 InstancedMesh 1개 = 드로우콜 5):

| 오브젝트 | 구성(전부 Box/Cylinder 조합) | 게임 역할 |
|---|---|---|
| 실험대 | 스테인리스 상판+서랍장, 상판 위 소형 플라스크 장식 | 표준 엄폐(허리 높이) |
| 시약장 | 키 큰 장+반투명 유리문+선반 병 3단 | 시야 차단(전신 높이) |
| 흄후드 | 상자+개구부+내부 어두운 공간 | 코너 엄폐 |
| 가스통 랙 | 실린더 2~3개 묶음+체인 | 원형 엄폐 |
| 폐시약 드럼 | 드럼통+경고 라벨 밴드 | 저높이 엄폐 |

콜라이더는 현행 AABB 유지(§v3 회전 불일치 문제를 피하려 **회전은 90° 단위만** 허용).

## G-3. 팀 기지·조립대 연출 (3순위)

- 산화팀 기지: 연소 실험 구역 콘셉트 — 버너 오브제 + 주황 포인트라이트 1 + 불꽃 파티클(소)
- 환원팀 기지: 전기분해 구역 콘셉트 — 전극조 오브제 + 시안 포인트라이트 1 + 기포 파티클(소)
- 조립대 패드: **홀로그램 부품 실루엣** — 아직 못 모은 부품을 반투명 팀색(opacity 0.25)으로
  미리 표시 → 뭘 더 모아야 하는지 공간적으로 보임(items.js makePartMesh 재사용, 교육적 명료성↑)
- 장착 순간: 실루엣→실물 전환 플래시 + 링 파동(effects.js 재사용)

## G-4. 캐릭터·전투 연출 (4순위)

- 복셀 캐릭터 변형: 고글·실험장갑·팀 두건(머리 블록 위 1복셀) — seed 기반 3종 조합
- **젖음 게이지 시각화**: gauge 비례로 몸통 머티리얼 컬러를 물색으로 lerp(맞을수록 젖어 보임 —
  원격 플레이어도 snap의 gauge로 동일 적용). 킬 순간 비눗방울 팝 파티클(비폭력 연출 유지)
- 발사 머즐: 무기 끝 물방울 스파클 3프레임, 달리기 먼지 퍼프(풀링 재사용)

## G-5. HUD 보조 (5순위, 선택)

- 미니맵: Canvas 2D 128px — 크레이트(미개봉만)·아군 점·조립대 마름모. 2Hz 갱신이면 충분
- 히트 방향 인디케이터: 피격 시 화면 가장자리 붉은 원호(공격 방향)

## 실행 계획·예상 규모

| 단계 | 파일 | 규모 | 비고 |
|---|---|---|---|
| G-1 | world/theme.js(신규)+arena.js 치환 | 중 | 시각 효과 대비 최저 비용 — 최우선 |
| G-2 | world/props.js(신규)+arena.js | 중 | 콜라이더 규칙 유지 필수 |
| G-3 | arena.js·items.js·effects.js | 소 | 홀로그램은 setAssembled 확장 |
| G-4 | voxel.js·players.js·effects.js | 중 | gauge→색 lerp는 players.js에 snap 연결 필요 |
| G-5 | ui/minimap.js(신규)+hud.js | 소 | 성능 여유 확인 후 |

- 성능 가드: 각 단계 후 `renderer.info.render.calls` 확인(150 초과 시 롤백),
  tablet 프리셋에서 포인트라이트·파티클 수 자동 절반
- 권장 진행: G-1+G-2를 에이전트 1회 투입으로 묶고, G-3~G-5는 브라우저 검증 후 2차 투입
