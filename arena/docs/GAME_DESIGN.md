# ChemVerse ARENA — 게임 설계서 v1.0

> 용인고 화학 수업용 팀 대전 게임. 픽셀건(복셀 슈터)의 '깃발 꽂기' 모드를 참조하되,
> 깃발 대신 **"실험 장치를 먼저 완성하고 미지 시료의 정체를 밝히는 팀이 승리"**하는 구조.
> 설계: Fable / 구현: Sonnet 하위 에이전트 / 검수: Fable. (ChemVerse 본편과 별개 앱, 같은 저장소 `arena/` 폴더)

---

## 0. 위치·배포

- 폴더: `ChemVerse/arena/` (본편 파일은 일절 수정하지 않음 — 보존)
- 배포: 기존 GitHub Pages 그대로 → 학생 URL `https://hlhl0000.github.io/chemverse/arena/`
- 스택: Three.js r170 CDN + 네이티브 ESM (본편과 동일, 빌드 도구 없음)
- 재사용: 본편의 물리 모델(`../js/experiments/idealgas/model.js`)은 **복사해서** 사용,
  net 어댑터·조이스틱·엔진은 포팅 후 수정. 디자인매뉴얼 v2.0 팔레트 계승.

## 1. 컨셉

- **장르**: 3D 복셀(블록형) 3인칭 슈터 + 아이템 수집 + 과학 탐구
- **인원**: 한 맵 4~10명, 2팀 — **산화팀(OX, 주황 #ff8a3d)** vs **환원팀(RE, 시안 #00b4d8)**
- **승리**: 맵에 흩어진 **실험 준비물(부품)**을 획득 → 팀 조립대에서 장치 완성 →
  측정값으로 **미지 기체의 분자량을 계산해 정체를 맞히면 즉시 승리**
- **무기**: 실험 기구(스포이트 물총·뷰렛 저격·시약 분무기·부피 플라스크 투척).
  피격은 "젖음 게이지"로 표현(비폭력) — 다 젖으면 실험복 교체(3초 리스폰)
- **연구윤리 룰**: 상대를 쓰러뜨리면 부품을 강탈할 수 있으나 **본인 점수 -5(연구윤리 위반)**.
  전투는 수단일 뿐 탐구가 목적임을 게임 규칙으로 체화

## 2. 게임 플로우

```
[로그인] 학년·반·이름 → [미션 선택] 카드 → [방 번호] 입력
   → roomKey = arena:{학년}-{반}-{방}-{미션id}  (전부 일치해야 같은 맵)
[대기실] 팀 선택(산화/환원, 자동 밸런스 제안) · 호스트(첫 입장자)가 [게임 시작]
   → 3초 카운트다운 → 맵 스폰
[매치] 제한시간 10분 — 부품 획득·조립·전투·퀴즈
[관찰 페이즈] 종료 후 완성 장치로 실제 측정 재현 (Phase C)
[결과] 점수판 → 다시하기 / 로비로
```

## 3. 매치 규칙 (Phase B 구현 대상)

### 3-1. 부품과 조립
- 미션마다 부품 목록 정의 (이상기체: 교과서 '물질과 에너지' 20쪽 준비물 6종 그대로 —
  온도계 · 휴대용 기체 통 · 주사기(100~200 mL) · 투명 튜브 관 · 전자저울 · 스탠드와 집게)
- 부품은 맵의 **공급 상자**(중앙 격전지에 다수, 외곽에 소수)에서 획득. 탭/근접으로 줍기
- 1인당 소지 2개 제한 → 팀 조립대(자기 진영 발광 패드)에 가서 장착
- 조립대에 전 부품 장착 = **장치 완성** → 측정 콘솔 활성화

### 3-2. 최종 퀴즈 (승리 조건)
- 장치 완성 팀에게 측정값 공개: 기체 질량 w(기체 통 질량 변화), 부피 V(주사기 눈금), 온도 T, 대기압 P
- `M = wRT/PV` 계산 → 보기(He 4 / N₂ 28 / O₂ 32 / CO₂ 44 / C₄H₁₀ 58 g/mol) 중 선택
- 정답 = **즉시 승리**. 오답 = 팀 -20점, 30초 재도전 잠금 (계산을 대충 하면 손해)
- 미지 기체는 매치 시드로 결정 — 매판 다름

### 3-3. 전투
- 기본 무기 스포이트 물총(3발, 재장전 1.5초). 공급 상자에서 뷰렛(장거리 정밀)·
  시약 분무기(근접 광역)·부피 플라스크(투척 포물선) 드랍
- 젖음 게이지 100. 0이 되면 3초 리스폰(자기 진영), 소지 부품 1개 드랍
- 처치자: 드랍 부품 획득 가능 + **점수 -5 (연구윤리 위반 로그가 킬피드에 표시)**

### 3-4. 점수 (제한시간 종료 시 승부 판정용)
| 행동 | 점수 |
|---|---|
| 부품 획득 | +10 |
| 조립대 장착 | +15 |
| 장치 완성 | +30 (팀 전체) |
| 퀴즈 정답 | 즉시 승리 |
| 퀴즈 오답 | -20 |
| 상대 처치 | **-5** (연구윤리 위반) |
- 제한시간 종료 시: 팀 총점 비교 → 승/패/무

## 4. 관찰 페이즈 (Phase C 구현 대상)

게임 종료 후 전원이 중앙 관찰 구역으로 이동, 승리 팀의 완성 장치로 실제 현상 재현:
주사기 피스톤이 밀리며 V 눈금 변화, 저울 수치 변화, T·P 표시 → M 계산 과정이
단계별 수식으로 표시 → 기체 정체 공개. 물리는 본편 `idealgas/model.js`(PV=nRT, 검증 완료) 복사본 사용.
라울 법칙 미션은 용액 증기압 내림 관찰로 동일 구조.

## 5. 네트워크·권한 모델

- 어댑터 3종(solo/supabase/wsrelay)은 본편 계약을 **v2로 확장**: 상태 채널(스로틀 8Hz) +
  임의 이벤트 채널 `send(type, payload)` / `on('msg')` 추가. 서버는 전부 "바보 파이프"
- **호스트 권한(host authority)**: 방의 첫 입장자(최소 joinTs, 동률 시 id 사전순)가 심판.
  시드 생성·게임 시작·(Phase B) 부품 획득 승인·킬 확정·점수·타이머 종료를 호스트가 판정해 브로드캐스트.
  호스트 이탈 시 로스터 차순위가 자동 승계(host migration)
- **트래픽 산정(정직)**: 10인 룸 · 8Hz · 10분 매치 = 전송 4.8만 · 팬아웃 포함 약 43만 msg/매치.
  Supabase 무료 티어(월 200만)로는 **월 4매치 규모** → 수업 본번은 **교내망 릴레이**(`arena/server/relay.js`,
  교사 PC) 필수, Supabase는 시연·과제·소규모용. `?net=wsrelay|supabase|solo` URL 파라미터로 강제 가능

## 6. 그래픽 방향

- **복셀 TPS**: 블록 조립형 캐릭터(머리·몸·팔다리 BoxGeometry, 걷기 애니메이션), 어깨너머 3인칭 카메라(붐 4m)
- 맵: 40×28 m 대칭 아레나. 팀 기지(조립대) 양끝, 중앙 격전지, InstancedMesh 엄폐 블록, 시드 기반 배치(전 클라이언트 동일)
- 팔레트: 디자인매뉴얼 v2.0 다크(#0e1117) 기반 + 팀 컬러(주황/시안, colorblind-safe 쌍)
- 폰트: **DungGeunMo(둥근모꼴)** 웹폰트 — 한글 픽셀 게임 감성 + 수치 가독성
- 성능 예산 본편 계승: 그림자맵 금지, 텍스처 금지(버텍스 컬러), 드로우콜 ≤150, pixelRatio ≤1.5, 30fps

## 7. 미션 레지스트리

미션 = 부품 목록 + 조립 규칙 + 측정 모델 + 퀴즈. `js/missions/registry.js` 스키마 참조.
1. **idealgas — "미지 기체를 밝혀라"** (1순위 완성): 위 §3 상세
2. **raoult — "증기 압력의 비밀"** (스텁): 밀폐 용기·마노미터·용매·비휘발성 용질·온도계·항온조를
   조립 → 증기압 내림 Δp = p°·x_용질 측정 → 용질 몰질량 계산해 정체 판별
