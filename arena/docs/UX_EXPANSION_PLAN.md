# ChemVerse ARENA — UX·퀄리티 확장 계획 (Phase U, 설계: Fable)

> 근거: 실전 대전 게임(모바일 FPS·Splatoon·Overwatch·Fortnite)의 확립된 UX 방법론을
> 태블릿 우선·교육용·비폭력(물 테마) 조건에 맞게 번안. U-1~U-5는 이번 세션 구현 완료,
> U-6 이후는 Opus 확장용 백로그 — 각 항목에 구현 파일·접점·주의사항을 명시해
> 재설명 없이 착수 가능하도록 작성.

## 공통 원칙 (전 항목 위반 금지)

- 신규 UI는 전부 **독립 모듈**(`ui/*.js`) — hud.js를 수정하지 않고 body 직속 오버레이로 부착,
  스타일은 모듈이 자체 `<style>` 주입(1회 가드). 파일 간 결합은 main.js 배선으로만.
- 성능 예산 유지: 텍스처·그림자 금지, DOM 오버레이는 pointer-events:none, 갱신 주기 명시
- z-index 대역: 캔버스 0 / 조작존 10 / 발사·버튼 12 / **신규 오버레이 11~15** / HUD 20 / 모달 40+
- 비폭력 연출: 피격·킬 피드백은 전부 물/거품 테마(붉은 피 금지 — 시안·블루 사용)

## 구현 완료 (이번 세션)

### U-1. 크로스헤어 + 히트마커 — `ui/reticle.js`
모바일 FPS 표준: 중앙 레티클 없이는 1인칭 조준 불가. 4틱+중앙점, 발사 시 확산(반동 표현),
킬 확정 시 X 히트마커 플래시. main.js 접점: `hud.onFire`→`pulseFire()`, `ev kill(shooter=나)`→`hitmark()`,
1인칭 블렌드≥0.6일 때만 표시.

### U-2. 피격 피드백 — `ui/feedback.js`
Overwatch식 피격 방향 원호 + 젖음 비네트. 공격자 방위는 `RemotePlayers.getPosition(shooter)`와
내 yaw로 화면각 계산(main.js `screenAngleTo`). 게이지 50 미만부터 화면 가장자리 물색 비네트가
점진 강화(저체력 경고). 솔로/공격자 이탈 시 방향 없는 전면 플래시로 폴백.

### U-3. 목표 웨이포인트 — `ui/waypoints.js`
Fortnite/CoD식 오프스크린 에지 마커. 3D 좌표를 카메라로 투영, 화면 밖이면 가장자리에
방향 화살표+거리(m) 표시. 규칙: 인벤 비었으면 "가까운 부품"(items.nearestPickup의 pos 필드),
인벤 있으면 조립대 강조. 학생이 "지금 뭘 해야 하는지"를 공간적으로 인지 — 교육용 핵심 UX.

### U-4. 미니맵 — `ui/minimap.js` (GRAPHICS_PLAN G-5 이행)
Canvas 2D 좌상단, 4Hz 갱신. 내 위치(방향 화살표)·아군 점·크레이트(미개봉)·드랍·조립대 마름모.
적군은 표시하지 않음(정보전 요소 유지 — 실전 관례). 데이터: `items.listMapPoints()`,
`RemotePlayers.listForMap()`(아군 필터는 main.js).

### U-5. 감도 설정 — `ui/settings.js` + `tps.setLookSensScale(k)`
대전 게임 필수 옵션 1순위 = 조준 감도. 우하단 기어 버튼 → 슬라이더(0.5~2.0x),
`localStorage('cv.sens')` 저장, 입장 시 자동 적용. tps.js는 배율 하나만 노출(0.3~3 클램프).

## 백로그 (Opus 확장용 — 우선순위순)

### U-6. 홀드 연사 + 리코일 킥
발사 버튼 pointerdown~up 동안 무기 fireRate로 연사(현재 탭 1발). 발사 시 카메라 pitch +0.5°
킥 후 복원. 접점: hud.js `.pb-fire`에 리스너 추가(기존 onFire와 중복 발사 주의 —
hud.onFire를 hold 시작으로 대체), tps.js에 `kick(rad)` 추가. combat.fire는 내부 쿨다운이
이미 있어(ammo/rate) 그대로 재사용.

### U-7. 히트 컨펌 강화 (심판 프로토콜 확장)
현재 피해는 피해자만 감지 → 사수는 킬 전까지 명중을 모름. referee.js `hp` ev에 `shooter`
필드 추가(하위호환: 기존 수신자는 무시) → 사수 화면에 히트마커·데미지 틱 표시.
**주의: AGENT_BRIEF_PHASE_B.md 프로토콜 문서도 함께 갱신할 것.**

### U-8. 킬캠·관전
사망 5초 동안 팀원 시점 관전(카메라를 팀원 위치로 lerp). RemotePlayers 위치 접근자(이미 추가됨)
재사용. 리스폰 카운트다운 HUD는 유지.

### U-9. 온보딩 코치마크
첫 입장(localStorage 플래그) 시 단계 안내: ①조이스틱 ②시선 드래그 ③상자 접근 자동 획득
④조립대 ⑤퀴즈. 각 단계는 해당 행동 감지로 자동 진행(설명 강요 금지 — 행동 유도).

### U-10. 절차 합성 사운드 (Phase D 선행)
WebAudio 순수 합성(에셋·CDN 불필요): 발사 '퓩', 명중 '퐁', 픽업 차임, 조립 클릭, 퀴즈 정답
아르페지오. `core/sfx.js` 신설, 볼륨/음소거는 U-5 설정 패널에 토글 추가.

### U-11. 접근성: 좌손 모드·색약 모드
좌손: 조이스틱/발사 버튼 좌우 미러(CSS 클래스 토글). 색약: 팀 색 OX 주황→적황 강조 외에
아이콘 형태 구분(산화=△, 환원=▽)을 이름표·미니맵·마커에 병기.

### U-12. 결과 화면 통계 보드
개인 기여 분해: 픽업/장착/퀴즈/킬/데스 카운트 표(교사 확인용). referee가 이미 pers를
집계하므로 ev end의 pers 스키마에 카운트 필드 추가 + hud.showResult 확장.

### U-13. 네트워크 보간 개선
RemoteEntry 지터 버퍼(100ms 지연 재생)와 외삽 상한(200ms) 명시화 — 태블릿 와이파이
패킷 몰림 시 원격 캐릭터 순간이동 방지. players.js 내부만 수정(계약 불변).

### U-14. 그래픽 후속 G-3·G-4 (GRAPHICS_PLAN 참조)
팀 기지 연출(버너/전극조+포인트라이트), 조립대 홀로그램 부품 실루엣(미획득 부품 표시 —
교육 명료성), 캐릭터 젖음 시각화(gauge→몸통 색 lerp, 원격은 snap gauge), 킬 비눗방울 팝.

## 구현 순서 권장

U-6·U-7(전투 손맛) → U-9(온보딩) → U-10(사운드) → U-14(그래픽) → U-8·U-12 → U-11·U-13.
각 단계 후 태블릿 실기기에서 30fps·드로우콜 확인. U-7만 프로토콜 변경이므로 멀티 2기기 검증 필수.
