  **이것은 실제 미커밋 변경이 아니라 샌드박스 마운트 플래핑이다.** 판별 근거:
  - `git diff --stat HEAD`가 이 5개 파일을 **삭제(-379줄)** 방향으로 보고 → bash 마운트가 Phase B **이전(더 작은)** 스냅샷을 들고 있어 최신 커밋(afd0797)과 비교되며 "지워진 것처럼" 보이는 것
  - 반대로 `git show afd0797:arena/js/main.js | wc -l = 417`, `git show afd0797:arena/css/arena.css | wc -l = 294` — **커밋된 내용은 최신 완본**이고, 호스트 Read도 main.js 418줄·arena.css 295줄로 완본 확인
  - 즉 **커밋(afd0797)과 호스트 파일이 진실이고, bash가 낡은 바이트를 본다.** 다음 세션에서 bash `git status`에 이 5줄이 또 떠도 **커밋하지 말 것**(빈/역방향 커밋 위험). 확정 판별: `git show HEAD:<경로> | wc -l`을 호스트 Read 줄수와 대조 → 일치하면 clean
- ⚠ **git 쓰기 명령(add/commit/push)은 샌드박스에서 상시 실행 불가.** 커밋·푸시는 v1~v3 내내 그랬듯 사용자 PC cmd에서 수행한다(이번 세션 `ce333a1`·`afd0797` 모두 사용자 cmd). 에이전트는 `git log`/`git show`/`git diff` 읽기만 수행하고 실측 결과를 문서화한다

## 3. 핵심 변경

### 3-1. Phase B 설계·프로토콜 확정 (Fable, `AGENT_BRIEF_PHASE_B.md`)

에이전트 투입 전 확정한 게임 규칙·네트워크 계약. **다음 세션이 전투/심판 코드를 읽기 전 필독.**

- **피격 판정** = 피해자 자가 판정(로컬 히트 감지) + **호스트 확정**(권위 서버 방식). 클라이언트는 신고, 호스트가 최종 승인
- **호스트 승계** = 2초 주기 snap 미러(`state.js` `lastSnap`)에서 Referee를 재구성(`referee.js` `fromSnap`). 호스트 이탈 시 새 호스트가 마지막 스냅으로 심판 상태를 복원
- **픽업·장착 자동** = 크레이트/드랍 근접 **2.0m**에서 자동 획득, 조립대(assembly zone) **반경 내**에서 자동 장착/조립. 버튼 조작 없음
- **크레이트 16개** = 부품 6종 × 2 + 무기 4(buret, spray, spray, flask). `loot.js`가 분배, `arena.js`가 배치
- **무기 4종 수치 동결**(combat.js `WEAPONS`): buret(뷰렛)·spray×2·flask(플라스크). flask는 포물선 투척, spray는 산탄. 이번 세션 수치 변경 금지 대상
- **점수표**: 부품 획득 +10 / 장착 +15 / 완성 +30 / 퀴즈 오답 -20 및 30초 잠금 / 킬 **-5(연구윤리 페널티)**
- **솔로에서도 심판 로컬 실행** — 상대가 없어도 Referee가 로컬 루프백(`__cvLocalReq`/`__cvLocalEv`)으로 돈다. 스모크 테스트 가능
